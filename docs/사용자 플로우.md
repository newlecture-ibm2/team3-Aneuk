[최종 최적화] 호텔 AI 서비스 통합 플로우

1. 진입 및 자동 식별 (Entry & Recognition)
액션: 고객이 객실의 QR 코드를 스캔.

시스템 동작: QR 데이터에서 room_id(예: 303)를 즉시 추출.

데이터 연동: 시스템이 내부 DB(또는 PMS)에서 해당 호수의 투숙객 성함(guest_name)을 가져옴.

1. 간편 환영 및 확인 (Welcome & One-tap Check)
화면 노출: "반갑습니다, 김철수 님! (303호)" 문구와 함께 [확인] 혹은 [서비스 시작] 버튼 노출.

액션: 사용자가 [확인] 버튼 클릭.

의의: 복잡한 인증 없이 "내 정보가 맞다"는 것만 인지시키고 즉시 입력 단계로 진입.

1. 요청 접수 및 AI 분석 (AI Processing)
액션: 사용자가 음성이나 텍스트로 자유롭게 요청.

예: "미니바 맥주 하나랑 수건 좀 더 가져다줘."

AI 동작: * 업무 지시화: {부서: 객실관리, 항목: 수건}, {부서: 식음료, 항목: 맥주} 형태로 데이터 구조화.

원문 보존: 사후 검증을 위해 "고객 발화 원문" 전체를 로그로 기록.

1. 업무 라우팅 및 기록 (Task Routing & Logging)
데이터 저장: Task 객체 생성 (호수, 이름, 요약 데이터, 원문 데이터, 상태값 포함).

자동 배분: 요약된 업무 데이터를 기반으로 해당 부서 직원의 단말기에 실시간 알림 전송.

1. 완료 피드백 (Closing)
액션: 부서 직원이 서비스 완료 후 시스템에 '완료' 입력.

피드백: AI가 고객에게 "요청하신 업무가 완료되었습니다"라고 안내하며 종료.

원문의 중요성: 별도의 본인 인증 과정이 없기 때문에, 나중에 결제 관련 이슈가 생겼을 때 **raw_message(원문 데이터)**가 유일한 증거가 됩니다. 따라서 DB에 원문을 저장하는 로직은 필수입니다.

데이터 구조 예시:

JSON
{
"task_id": "T1001",
"room_no": "303",
"guest_name": "김철수",
"summary": "수건 1, 맥주 1", // 직원이 볼 데이터
"raw_text": "맥주 하나랑 수건 좀 더...", // 검증용 원문
"status": "COMPLETED"
}

**~~~~**보안 단계를 덜어내니 로직이 훨씬 간결해졌습니다. 이제 이 플로우를 바탕으로 백엔드 API나 DB 스키마(예: Task 테이블)를 구성하시면 될 것 같습니다. 더 구체적인 코딩 로직이나 DB 설계가 필요하시면 말씀해 주세요!
