## 2. 핵심 루프

```
┌─────────────────────────────────────────────────────────────────────┐
│                        아늑 핵심 루프                                 │
│                                                                      │
│  ① 고객 요청 입력 (다국어 텍스트 / 빠른 버튼 / QR 접속)               │
│       │                                                              │
│       ▼                                                              │
│  ② AI 분석 (Intent + Entity + Confidence Score)                      │
│       │                                                              │
│       ├─ confidence ≥ 0.7 → ③ 태스크 자동 생성                       │
│       │                                                              │
│       └─ confidence < 0.7 → ⑥ Fallback (관리자/프론트 에스컬레이션)   │
│                                                                      │
│  ③ 태스크 자동 생성 (부서 매핑 + 상태: PENDING)                       │
│       │                                                              │
│       ▼                                                              │
│  ③-1. 하이브리드 분기                                                │
│       ├─ [단순 요청] confidence ≥ 0.8 → ④ 자동 부서 배분              │
│       └─ [복잡 요청] confidence < 0.8 → 관리자 컨펌 대기 → ④          │
│                                                                      │
│  ④ 부서 배분 → 담당 직원 알림 (상태: ASSIGNED)                        │
│       │                                                              │
│       ▼                                                              │
│  ⑤ 직원 수행 + 완료 피드백 (상태: IN_PROGRESS → COMPLETED)            │
│       │                                                              │
│       ├─→ 추가 요구 발견 → ② AI 재분석 → 새 태스크 생성               │
│       ├─→ 특이사항 → 관리자 에스컬레이션                               │
│       └─→ 완료 데이터 → ⑧ RAG 학습                                   │
│                                                                      │
│  ⑥ Fallback (AI 실패/모호 요청 → 관리자/프론트 수동 처리)             │
│                                                                      │
│  ⑦ 고객 실시간 진행 상태 확인                                         │
│     (접수됨 → 담당자 배정 → 처리중 → 완료)                             │
│                                                                      │
│  ⑧ RAG 학습 → 서비스 품질 지속 향상                                   │
│                                                                      │
│  ⑨ 관리자: 전체 태스크 모니터링 + 수동 분배 + 컨펌                     │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

## 4. MVP 핵심 기능 (11개)

---

### F1. 고객 요청 입력 (Guest Chat)

| 항목 | 내용 |
|------|------|
| **설명** | 투숙객이 자연어로 요청을 입력하는 인터페이스 |
| **입력 방식** | ① 텍스트 (다국어) ② 빠른 요청 버튼 |
| **접근 방식** | 객실 QR코드 스캔 → PWA 즉시 접속 (앱 설치 불필요) |
| **다국어** | 한국어, 영어 (MVP) / 일본어, 중국어 (Phase 2) |
| **QR 연동** | URL 파라미터로 객실번호 자동 입력 (`?room=302`) |

```
투숙객 → QR 스캔 → 언어 선택 → 객실번호+이름 인증 → AI 대화 화면
```

---

### F2. AI 의도 분석 (Intent + Entity + Confidence)

| 항목 | 내용 |
|------|------|
| **설명** | 고객의 자연어 요청을 구조화된 데이터로 변환 |
| **핵심 기술** | Gemini API (Intent Classification + Entity Extraction) |
| **출력 데이터** | 초경량 Task-Code (`[부서]\|[긴급도]\|[분류코드]\|[상세항목]\|[수량]`) + confidence score |
| **다국어 처리** | One-pass 프로세싱: 언어 감지 없이 단일 프롬프트로 번역+의도+식별자 동시 추출 → 한국어 기준 코드 변환 |

```
입력: "Can I get two towels please?" (영어, confidence: 0.92)

출력: HK|NORM|REQ_ITEM|TOWEL|2
→ 부서: HOUSEKEEPING / 긴급도: NORMAL / 분류: 비품요청 / 품목: 수건 / 수량: 2

AS-IS (약 40토큰): "담당 부서: 객실 관리팀, 긴급도: 보통, 분류: 비품요청, 품목: 수건 2장"
TO-BE (약  8토큰): HK|NORM|REQ_ITEM|TOWEL|2 (80~90% 토큰 절감)
```

**Confidence 기반 분기:**

| 조건 | 동작 |
|------|------|
| `confidence ≥ 0.8` | 자동 태스크 생성 + 자동 부서 배분 |
| `0.7 ≤ confidence < 0.8` | 태스크 생성 + **관리자 컨펌** 후 배분 |
| `confidence < 0.7` | **Fallback** → 프론트 데스크 에스컬레이션 |

### F3. 태스크 자동 생성

| 항목 | 내용 |
|------|------|
| **설명** | AI 분석 결과를 업무 단위(Task)로 변환하여 시스템에 등록 |
| **자동 생성 항목** | 제목(AI 요약), 카테고리, 부서, 긴급도, 객실, 상세 내용, 상태 |
| **초기 상태** | `PENDING` (접수됨) |
| **원문 보존** | 고객 원문(rawMessage)을 반드시 함께 저장 (디버깅/오류 확인용) |

---

### F4. 부서 배분 + 직원 알림

| 항목 | 내용 |
|------|------|
| **설명** | 생성된 태스크를 담당 부서로 자동 전달하고 직원에게 알림 |
| **배분 로직** | AI가 판별한 `department` 코드 기반 자동 라우팅 |
| **알림 방식** | WebSocket 실시간 알림 (직원 대시보드) |
| **상태 변경** | `PENDING → ASSIGNED` |
| **미분류 시** | 기본값: `FRONT_DESK` (Fallback 부서) |

> **MVP에서는 부서 단위 라우팅**만 구현. Smart Assign(개인 배정)은 Phase 2에서 추가.

---

### F5. 직원 업무 처리 + 상태 관리

| 항목 | 내용 |
|------|------|
| **설명** | 직원이 태스크 상태를 업데이트하고 응답 메시지를 전송 |
| **상태 전이** | `PENDING → ASSIGNED → IN_PROGRESS → COMPLETED` |
| **직원 응답** | 텍스트 메시지 → 투숙객 언어로 자동 번역 후 전달 |
| **부서 전달** | 잘못된 부서로 라우팅된 경우 → 다른 부서로 전달 (사유 입력) |
| **내부 메모** | 투숙객에게 보이지 않는 직원 간 메모 기능 |

**상태 Enum:**

```
PENDING      → 접수됨 (AI 생성 직후)
ASSIGNED     → 부서 배분 완료
IN_PROGRESS  → 처리 중 (직원이 수락)
COMPLETED    → 완료
ESCALATED    → 관리자/프론트 이관 (Human-in-the-loop)
TRANSFERRED  → 다른 부서로 전달됨
```

---

### F6. Fallback 처리 (AI 실패 대응)

| 항목 | 내용 |
|------|------|
| **설명** | AI가 의도 파악에 실패하거나 confidence가 낮은 경우의 처리 |
| **트리거** | intent 미확정 / confidence < 0.7 / 필수 entity 누락 |
| **동작** | 태스크를 `ESCALATED` 상태로 생성 → 프론트 데스크/관리자에게 원문 전달 |
| **고객 안내** | "요청을 직원에게 전달했습니다. 곧 연락드리겠습니다." |

**추가 처리 (Clarification):**

```
모호한 요청 → AI가 되물어봄:
  "무엇을 가져다 드릴까요?"
  🛁 수건 | 💧 생수 | 🩴 슬리퍼 | 🧴 어메니티
```

### F7. 고객 요청 진행 상태 추적

| 항목 | 내용 |
|------|------|
| **설명** | 투숙객이 자신의 요청 진행 상태를 실시간 확인 |
| **표시 상태** | 접수됨 → 담당 부서 배정 → 처리 중 → 완료 |
| **다국어** | 상태 메시지를 투숙객 언어로 표시 |
| **실시간** | WebSocket으로 상태 변경 즉시 반영 |

```
투숙객 화면:
  ● 접수됨         ✅ 완료 (10:01)
  ● 담당자 배정     ✅ 완료 (10:02)  "하우스키핑 담당"
  ● 처리 중        🔄 진행중
  ○ 완료           ⏳ 대기
```

### F8. 관리자 대시보드 + 수동 분배 + 컨펌

| 항목 | 내용 |
|------|------|
| **설명** | 전체 태스크를 실시간 모니터링하고 수동 개입하는 화면 |
| **실시간 태스크 보드** | 상태별 필터 (PENDING / IN_PROGRESS / COMPLETED / ESCALATED) |
| **수동 재배정** | AI 배정을 오버라이드하여 다른 부서/직원에게 변경 |
| **수동 태스크 추가** | 관리자가 직접 태스크 생성 후 배정 |
| **컨펌 대기열** | AI 불확실 건 + 직원 에스컬레이션 건 모아보기 |
| **긴급 건 강조** | 미처리 긴급 요청, 시간 초과 건 빨간색 하이라이트 |

**하이브리드 모델 (Confidence 기반 선택적 자동화):**

| 조건 | 자동화 수준 | 설명 |
|------|------------|------|
| `confidence ≥ 0.8` | **완전 자동** | 태스크 자동 생성 + 자동 부서 배분 (관리자 컨펌 불필요) |
| `0.7 ≤ confidence < 0.8` | **반자동 (컨펌)** | 태스크 생성 + **관리자 컨펌** 후 부서 배분 |
| `confidence < 0.7` | **수동 (에스컬레이션)** | Fallback → 프론트 데스크 에스컬레이션 (관리자 수동 처리) |

> MVP부터 **Confidence Score 기반 선택적 자동화**를 적용합니다. AI가 확신하는 요청(≥0.8)은 즉시 자동 처리하고, 불확실한 요청만 관리자가 컨펌합니다.

### F9. RAG 기반 정보 응대 + 학습

| 항목 | 내용 |
|------|------|
| **설명** | 호텔 정보(FAQ)를 RAG로 응대 + 요청 데이터를 학습하여 품질 향상 |
| **FAQ 응대** | 조식 시간, 와이파이, 편의시설 등 반복 질문에 AI 즉답 |
| **핵심 기술** | RAG (pgvector) — 호텔 정보 벡터 DB에서 검색 후 답변 생성 |
| **할루시네이션 방지** | 유사도 < 0.7이면 "정보 없음" 처리 → 프론트 연결 안내 |
| **학습 데이터** | 완료된 태스크의 분류/처리 이력을 관리자 컨펌 후 지식에 반영 |

```
[데이터 수집] 고객 요청 + 직원 처리 + 평가
       ↓
[AI 패턴 추출] 자주 발생하는 요청, 최적 처리 방법
       ↓
[관리자 컨펌] 승인된 것만 RAG 등록
       ↓
[품질 향상] 분류 정확도↑, 예상 시간 정확도↑
```

### F10. 직원 추가 요구 등록 → AI 재분배

| 항목 | 내용 |
|------|------|
| **설명** | 직원이 업무 중 발견한 추가 필요사항을 등록하면 AI가 재분석하여 태스크 생성 |
| **동일 파이프라인** | 직원 입력도 F2(AI 분석) → F3(태스크 생성) → F4(부서 배분) 동일 흐름 |
| **관리자 에스컬레이션** | 직원 → "외부 업체 수리 필요" → 관리자에게 즉시 알림 |

```
Case: 하우스키핑 직원이 청소 중 전구 고장 발견
  → "302호 전구 나감" 입력
  → AI 분석: 시설/유지보수 > 조명
  → 시설관리 부서에 새 태스크 자동 생성
  → 원래 직원에게도 처리 상태 추적 가능
```

### F11. 교대 인수인계 자동화 (Shift Handover Automation)

| 항목 | 내용 |
|------|------|
| **설명** | 교대 시간에 다음 근무자가 즉시 파악해야 할 핵심 사항을 AI가 자동으로 취합·요약하여 브리핑 |
| **접근 화면** | 관리자 대시보드(A-02) 내 '교대 인수인계' 탭 |
| **데이터 소스** | 백엔드에서 해당 교대 시간(Shift) 내 태스크를 자동 필터링 (Zero-Data-Entry) |

**인수인계 대상 태스크 자동 취합 (3가지 그룹):**

| # | 그룹 | 필터 조건 |
|---|------|-----------|
| ① | 잔여 업무 | 아직 완료되지 않은 태스크 (`PENDING`, `IN_PROGRESS`) |
| ② | 특이사항 | 처리 완료되었으나 **관리자 컨펌/에스컬레이션**을 거친 건 (`confidence < 0.7` 또는 `ESCALATED` 이력) |
| ③ | 주요 처리 내역 | 정상 자동 분배되어 완료된 건 중 **주요도가 높은 핵심 태스크** (`URGENT`, `HIGH`) |

**AI 브리핑 요약:**

*   취합된 태스크 JSON을 Gemini API에 One-pass로 주입하여 **3~5줄의 자연어 브리핑**을 자동 생성.
*   *예시: "302호 전구 교체 건이 3시간째 미처리 상태로 우선 확인 필요합니다. **[주요 처리]** 금일 VIP 501호 누수 긴급(URGENT) 건은 시설팀에서 10분 만에 조치 완료했습니다. **[특이사항]** 405호에서 규정에 없는 가습기 요청이 있어 프론트 창고 확인 후 예외 대여 처리했습니다."*

```
교대 시점:
  ① 백엔드가 미처리 + 에스컬레이션 + URGENT/HIGH 태스크 자동 취합
  ② 취합된 JSON → Gemini API One-pass 요약
  ③ 관리자 대시보드 '인수인계' 탭에 브리핑 텍스트 + 태스크 리스트 표시
  ④ 다음 근무자가 탭을 열면 즉시 상황 파악 완료
```
