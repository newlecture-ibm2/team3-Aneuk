호텔 AI 서비스의 핵심은 고객의 요청이 데이터화된 이후, 이를 **관리하는 관리자(Admin/Front)**와 **현장에서 움직이는 실무자(Staff)** 사이의 유기적인 연결입니다.

제공해주신 이미지의 **"정보 허브 + 작업 라우터"** 개념을 바탕으로 두 역할의 플로우를 정리해 드립니다.

# 1. 프론트 및 관리자 (Admin/Front) 플로우

**역할:** 전체 업무 모니터링, 예외 상황 처리, 데이터 통계 관리

### 1. **실시간 대시보드 모니터링**

AI가 생성한 모든 `Task`를 실시간 리스트로 확인합니다.

**상태 필터링:** '대기 중', '처리 중', '지연됨', '완료' 상태를 한눈에 파악합니다.

### 2. **예외 상황 개입 (Human-in-the-loop)**

**AI 해석 실패 시:** AI가 의도를 파악하지 못해 `Unknown`으로 분류한 항목을 직접 확인하여 수동으로 부서를 지정합니다.

**지연 업무 처리:** 특정 부서에서 업무 처리가 10분 이상 지연될 경우 알람을 받고, 해당 부서에 독촉하거나 인력을 재배치합니다.

### 3. **대화 원문(Raw Data) 검증**

고객과 실무자 사이에 분쟁이 발생하거나 요청 내용이 불확실할 때, AI가 저장한 **고객 발화 원문**을 조회하여 정확한 근거를 확인합니다.

### 4. **서비스 품질 통계 분석**

부서별 평균 처리 시간, 가장 많이 들어오는 요청(수건, 생수 등)을 분석하여 호텔 운영 정책(비품 재고 증설 등)에 반영합니다.

# 2. 현장 실무자 (Staff) 플로우

**역할:** 할당된 업무 수행, 실시간 상태 업데이트

### 1. **개별 작업 알림 수신**

담당 부서(예: 객실관리팀)에 해당하는 업무만 스마트폰이나 스마트워치로 푸시 알림을 받습니다.

**화면 구성:** `[303호 / 수건 2개 / 즉시]` 처럼 **AI가 요약한 핵심 지시 사항**이 최상단에 노출됩니다.

### 2. **업무 수락 및 상태 전환**

리스트에서 업무를 선택하고 **[수락/이동 중]** 버튼을 누릅니다. (이때 프론트 대시보드와 고객에게 "담당자가 이동 중입니다"라고 상태가 자동 공유됩니다.)

### 3. **상세 내용 확인 (필요 시)**

요약된 내용만으로 부족할 경우, 클릭하여 **고객의 요청 원문**을 확인합니다. (예: "애기가 물을 쏟아서 수건이 급해요"라는 맥락을 보고 더 빨리 이동 결정)

### 4. **업무 완료 및 보고**

현장에서 업무를 마친 후 **[완료]** 버튼을 누릅니다.

필요하다면 특이사항(예: "재고 부족으로 1개만 전달함")을 짧게 메모하거나 사진을 찍어 첨부합니다.

### 5. **피드백 확인**

고객이 남긴 간단한 만족도나 감사 인사를 확인하며 업무 동기부여를 얻습니다.

# 3. 관리자 vs 실무자 플로우 비교 요약

**구분관리자 (Admin)실무자 (Staff)주요 목적**전체 흐름 관리 및 리스크 방지할당된 개별 임무의 완수**데이터 뷰**전체 부서의 모든 Task 리스트본인 부서에 할당된 Task 리스트**핵심 액션**업무 강제 배정, 지연 관리, 통계 조회업무 수락, 진행 상태 변경, 완료 보고**AI 활용**업무 자동 배정 결과 모니터링정제된 요약 지시서 기반 즉시 출동

# 4. 개발 설계 시 고려할 아키텍처 포인트

- **Websocket/Push Notification:** 실무자 앱은 실시간성이 생명이므로, API 폴링 방식보다는 **Websocket**이나 **FCM(Firebase Cloud Messaging)**을 통한 실시간 알림 구조가 필수입니다.
- **Role-Based Access Control (RBAC):** 로그인한 계정의 권한에 따라 관리자 페이지와 실무자 페이지의 접근 권한을 엄격히 분리해야 합니다.
- **Event-Driven:** 실무자가 '완료' 버튼을 누르는 이벤트가 발생하면 -> AI가 고객에게 알림을 보내고 -> 관리자 대시보드가 갱신되는 **이벤트 중심 설계**가 효율적입니다.

이제 고객(Client), 관리자(Admin), 실무자(Staff)를 잇는 3각 플로우가 완성되었습니다. 이 흐름 중에서 특별히 더 상세한 UI 시나리오나 기능이 필요한 부분이 있을까요?
