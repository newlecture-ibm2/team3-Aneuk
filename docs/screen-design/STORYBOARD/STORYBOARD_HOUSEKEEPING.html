Description

Summary.

  • 하우스키핑 부서 전용 칸반(Kanban) 보드 대시보드
  • 고객 요청 상태(대기 중/진행 중/완료됨) 직관적 시각화
  • 관리자가 현장 스태프를 직접 지정하여 업무를 할당하는 흐름(Flow)
1

상태별 칸반 보드 (Columns)

· 대기 중, 진행 중, 완료됨 등 처리 단계에 따라 카드를 분류

· 상단에 해당 상태의 총 요청 건수를 숫자로 표시
2

요청 카드 정보

· 객실 번호 (#1204, #502 등)

· 우선순위 배지 (MEDIUM, HIGH 등)

· 요청 분류(수건 교체, 침구류 정리 등) 및 상세 내용 노출
3

직원 배정 액션

· 아직 업무가 할당되지 않은 요청 건에 대해 담당자를 배정하는 버튼
4

스태프 리스트 드롭다운 (Popup)

· '직원 배정' 클릭 시 노출되는 선택 메뉴

· 현재 근무 중인 하우스키핑 소속 직원(매니저, 현장 스태프) 목록 제공
5

배정된 담당자 정보 표시

· 담당자가 배정되면 카드 내부에 프로필 아이콘과 이름(예: 이영희)을 명확하게 노출
6

담당자 변경 액션

· 담당자가 배정된 이후에는 버튼 텍스트가 '담당자 변경'으로 전환되어, 업무 재할당을 지원
🎯 Event Description
E1 직원 배정 드롭다운 오픈

· 트리거: 요청 카드의 하단 '직원 배정' 텍스트 버튼 클릭

· 동작: 활성화된 직원 리스트(Floating Menu) 노출

· API: GET /api/staff/available?dept=HOUSEKEEPING
E2 담당자 할당 완료

· 트리거: 드롭다운 내 특정 스태프 선택 (클릭)

· 동작: 할당 API 호출 후 팝업 닫힘. 카드에 담당자 정보 UI 반영 및 상태 변경(대기중 -> 진행중 등 필요 시)

· API: PATCH /api/requests/{id}/assignee (body: staffId)

Check Point

배정된 직원의 디바이스(앱/웹)로 실시간 푸시 알림(FCM) 발송 처리 필요

다른 관리자가 동시에 배정할 경우를 대비하여 Optimistic UI 처리 또는 DB Lock 고민
