Description

Summary.

  • 고객 요청이 올바른 부서로 라우팅 되었는지 추적하는 AI 엔진 로깅 화면
  • 응답 지연 및 토큰 소모량을 파악하여 프롬프트 최적화 지표로 활용
  • 모달창을 통해 모델 입출력(Raw Data) 원문을 그대로 열람하여 디버깅 지원
1

핵심 지표 (Top KPIs)

· 평균 응답 속도: AI가 요청을 분석하고 응답하는데 걸리는 전체 평균 시간

· 누적 소모 토큰: 비용 관리를 위한 토큰 사용량 집계

· AI 라우팅 성공률: AI가 예외 처리(Fallback) 없이 스스로 부서를 할당한 비율
2

세부 접속 로그 리스트

· 발생 시간별로 개별 고객 요청(미리보기)을 리스트업

· 지연시간(Latency)이 비정상적으로 긴 경우 붉은색 SLOW 태그를 붙여 시각적으로 경고
3

상세 보기 (Detail Button)

· 클릭 시 해당 요청의 AI 처리 원문을 확인할 수 있는 하단의 상세 모달창(5번) 노출
4

요청별 상세 메트릭 (Modal)

· 해당 단일 요청에 사용된 입력/출력 토큰, 처리 시간, 사용된 모델(gemini-1.5-flash 등) 표기
5

RAW DATA 모니터링 (Modal)

· RAW PROMPT: AI에게 전달된 시스템 프롬프트 및 사용자 발화 원문

· RAW RESPONSE: AI가 백엔드로 반환한 최종 JSON 형태의 데이터 포맷. 의도(intent), 부서(domain), 품목(items) 추출 내역을 그대로 확인 가능하여 디버깅에 필수적임
🎯 Event Description
E1 로그 상세 모달 진입

· 트리거: 로그 리스트 우측의 '상세 보기' 버튼 클릭 (3번 마커)

· 동작: 백그라운드 딤 처리 후 해당 요청 번호(request_id)의 RAW 데이터 모달창 팝업

· API: GET /api/admin/logs/{requestId}

Check Point

민감한 고객 정보(PII)가 프롬프트 원문에 포함될 수 있으므로, 해당 화면은 최고 관리자 권한만 접근 가능하도록 설정해야 함

SLO(예: 응답시간 2000ms 초과) 위반 시 슬랙 알림 연동 고려
