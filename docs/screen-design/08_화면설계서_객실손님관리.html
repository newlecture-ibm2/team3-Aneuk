손님 추가 (체크인)

✅ 체크인 완료 시 모바일 키(QR) URL이 자동 발급됩니다.

✨ UI 뷰: 손님 추가 팝업

리스트 우측 상단의 '+ 손님 추가 (체크인)' 버튼을 클릭하면 화면 전체가 어두워지며 중앙에 나타나는 모달(Modal) 팝업입니다.

메인 화면 위에 독립된 레이어로 배치되며, 모달 바깥 영역을 클릭하거나 취소 버튼을 누르면 팝업이 닫힙니다.

Description
1객실 손님 관리 (가상 PMS 역할)
· 개발 및 테스트 목적으로 손님 데이터를 수동 생성하는 숨김/개발자용 메뉴
· 실제 PMS(호텔운영시스템) 연동 전, 객실 할당 역할을 시뮬레이션
2+ 손님 추가 (모달)
· 객실 번호, 이름, 체크인 일시, 체크아웃 일시를 수동으로 설정할 수 있는 모달 팝업
· 프론트 연동 없이 임의의 투숙 데이터를 생성하기 위한 목적
· 체크인 시 투숙객 앱(G-01)에 접근할 수 있는 모바일 키(QR 코드) URL이 발급되며, PIN 번호는 사용하지 않음 (화면에 표시되지 않음)
3객실 손님 리스트
· 리스트에는 현재 '투숙 중'인 손님 데이터만 표시됨
· 체크아웃 예정: 손님의 체크아웃 예정 날짜 표시
· ✏️ (수정): 손님 정보 편집
· 🗑️ (체크아웃/삭제): 클릭 시 체크아웃 처리되며, 데이터베이스에서 Hard Delete(완전 삭제) 되어 리스트에서 사라짐
🎯 Event
E1체크아웃 (Hard Delete)
· API: DELETE /admin/dummy/guests/{id}
· 휴지통(🗑️) 아이콘 클릭 시, 해당 게스트 정보가 Hard Delete 되며 더 이상 투숙객 앱에 로그인할 수 없음
💡 Check Point
C1본 화면은 오직 개발 과정의 프론트/앱 연동 테스트를 위한 가상 PMS 화면입니다.
