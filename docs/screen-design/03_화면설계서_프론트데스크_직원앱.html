객실: 1502호

✨ UI 뷰: 요청 상세 / 수동 재배정 모달

에스컬레이션 카드를 클릭하면 중앙에 나타나는 모달(Modal) 팝업입니다. 공통 모달 컴포넌트를 재사용할 예정입니다.

AI가 잘못 분류한 부서를 정정하거나 특정 직원에게 수동 배정할 수 있으며, 모달 바깥 영역을 클릭하거나 닫기 버튼을 누르면 팝업이 닫힙니다.

Description
1 프론트 데스크 메뉴
· 기존 프론트 데스크 화면과 동일한 사이드바 위치
2 승인 대기 (에스컬레이션) 탭
· 현장 직원의 권한 밖이거나 AI가 관리자 확인이 필요하다고 판단한 요청이 모이는 곳
· 기존 '대기 중인 요청' 탭 앞에 최우선으로 배치
3 에스컬레이션 카드
· 좌측에 빨간 강조선(border-left)으로 시각적 구분
· 승인/반려 버튼으로 즉각 결재 처리 가능
4 승인 / 반려 액션
· 승인 시 해당 태스크가 '대기 중' 탭으로 이동하여 직원 배정 가능 상태로 전환
· 반려 시 사유를 입력하고 요청자에게 안내
5 상세 / 수동 재배정 모달
· 카드 클릭 시 노출 (공통 모달 컴포넌트 사용 예정)
· AI가 잘못 분류한 부서를 정정하거나 특정 직원에게 수동 배정 가능
🎯 Event
E1 승인 클릭
· 에스컬레이션 건 승인 → '대기 중' 탭으로 이동
· API: PUT `/staff/requests/{id}/approve`
E2 반려 클릭
· 에스컬레이션 건 반려 → 사유 입력 후 요청자 안내
· API: PUT `/staff/requests/{id}/reject`
E3 카드 클릭 → 상세 모달
· 요청 원문, 부서 재배정, 담당자 수동 지정이 가능한 모달 팝업 노출
💡 Check Point
C1 WebSocket 연동을 통한 에스컬레이션 건 실시간 알림 필수
C2 관리자(ROLE_ADMIN) 권한이 있는 사용자만 승인/반려 버튼이 활성화되어야 함
