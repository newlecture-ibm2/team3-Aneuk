Description

Summary.

  • 게스트 챗봇의 7가지 상태를 한 화면에 종합 정리
  • AI Studio 시뮬레이터 실제 코드 기준 UI 재현
  • 각 상태별 컬러 레이블로 시각 구분
  • GNB 헤더 제외, 챗봇 영역만 표시
1 채팅 서비스 헤더
💬 아이콘 + "컨시어지 서비스" + "AI 퍼스널 어시스턴트"
2 ① Welcome (QR 첫 화면)
· 인사 문구 + 예시 안내
· QR 스캔 직후 자동 표시
3 ② Idle (입력 대기)
· Quick 버튼: 수건/어메니티/클리닝/체크아웃
· Pill 형태, 클릭 시 자동 메시지 전송
4 ③ Thinking (처리 중)
· 3-dot typing indicator 애니메이션
· AI 응답 생성 중 표시
5 ④ Success (응답 완료)
· 확인 메시지 + 상태 카드(progress bar)
· 접수 완료 → 처리 중 → 완료 단계 표시
6 ⑤ Clarification (추가 정보)
· AI 되물음 + Quick 버튼(필수)
· 예: "몇 개?" → [1개][2개][3개]
7 ⑥ Fallback (실패)
· 프론트 데스크 전달 안내
· 노란 보더 강조, 📞 아이콘
8 ⑦ Progress + Feedback
· 상태 카드: 100% progress, 완료 체크
· 피드백 카드: 별점 + 보내기 버튼
🎯 Event Description
E1 Quick 버튼 클릭
· 버튼 텍스트가 메시지로 자동 전송
· API: POST /api/chat/message
E2 메시지 전송 → Thinking 전환
· 전송 즉시 typing indicator 표시
· AI 응답 수신 시 indicator 제거 + 버블 렌더
E3 상태 카드 실시간 업데이트
· WebSocket/SSE로 progress 수신
· 접수→처리중→완료 단계별 바 갱신
E4 Clarification Quick 버튼 응답
· 수량 버튼 클릭 → 확정 메시지 전송
· 이후 Success 상태로 전환
E5 Fallback 에스컬레이션
· NLU confidence 미달 → 프론트 전달
· 2회 연속 시 직원 연결 제안
E6 피드백 제출
· 별점 선택 + 보내기 클릭
· API: POST /api/chat/feedback

Check Point

· 인증: 객실번호 + 이름 (Guest 세션, JWT 불필요)
· 챗봇 상태 순환: Welcome → Idle → Thinking → Success/Clarification/Fallback → Progress → Feedback
· Quick 버튼: Idle/Clarification 상태에서 표시
· 상태 카드: Success 시 생성, 실시간 업데이트
· Fallback: 프론트 데스크로 자동 에스컬레이션
· 파이프코드: DOMAIN|SEVERITY|ACTION|ITEM|QTY
