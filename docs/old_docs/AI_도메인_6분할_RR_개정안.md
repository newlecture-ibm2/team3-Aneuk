# 아늑(Anook) AI 설계 및 개발 R&R 개정안 (도메인 주도 6분할)

## 📌 변경 배경 (진정한 병렬 개발을 위한 R&R 재편)
이전 문서의 '기술 중심 릴레이 분할 방식(1번이 끝나야 2번이 진도 나가는 방식)'은 팀원 간 심각한 **병목 현상**을 유발합니다. 
이를 해결하기 위해 작업을 **6대 호텔 부서(도메인)** 기준으로 수직 분배합니다. 

단, **기존에 기획했던 6가지 핵심 AI 기술 스펙은 버리지 않습니다.** 
각 부서를 전담하는 담당자가 자신의 부서 성격에 가장 밀접한 **핵심 기술 스펙 1가지를 "메인"으로 개발 (Tech Lead)** 하여 다른 부서 담당자들에게 공유(Serve)하는 형태의 이상적인 병렬 구조입니다.

---

## 🗂️ 6개 부서(도메인) 및 매칭 기술 스펙

### 1. 🚨 프론트 데스크 (Front Desk) + 🛡️ NLU 예외 제어 (Safety & Fallback)
* **담당 도메인:** 메인 라우팅(의도 찾기), 체크인/체크아웃, 분류 불가 요청(Fallback)
* **담당 핵심 기술 스펙:** 
  * PII(개인정보) 정규식 마스킹 (프롬프트 전송 전 선제적 방어)
  * 미등록 품목(`ETC`) 발생 시 백엔드에서 Confidence Score 강등 및 라우팅 에스컬레이션 로직
* **적합한 성향 (Persona): "숲을 보는 아키텍트 & 깐깐한 방어자"**
  * 시스템이 예측 불가능한 질문이나 보안 공격에 터지지 않도록 엣지 케이스를 찾아내고 방어하는 구조적 논리가 필수적입니다.

### 2. 🛏️ 하우스키핑 (Housekeeping) + ⚙️ NLU Task 자동 분류 (해피 패스)
* **담당 도메인:** 수건, 어메니티, 청소 등 정형화된 비품 수량 요청 처리
* **담당 핵심 기술 스펙:** 
  * 다국어 One-pass 프롬프트 최적화 (언어 감지+번역+분류 통합)
  * 자연어를 `HK|NORM|TOWEL|2` 형태로 강제 변환하는 토큰 및 포맷 정규화
* **적합한 성향 (Persona): "원리원칙주의자 & 디테일 장인"**
  * 융통성보다는 오차 없는 "정확성"과 "포맷 유지"에 희열을 느끼며, 규격화된 파싱 로직이나 정규표현식에 능합니다.

### 3. 🍽️ 식음료/룸서비스 (F&B) + 🔍 RAG Inference (검색)
* **담당 도메인:** 레스토랑 시간, 메뉴판, 추가 비용 등 지식 룰이 엄격한 응대
* **담당 핵심 기술 스펙:** 
  * `pgvector` 기반 임베딩 및 유사도 검색 API 개발
  * 유사도 0.7 미만 시 "정보 없음"으로 Fallback 하는 방어 로직 (환각 방지)
* **적합한 성향 (Persona): "치밀한 데이터베이스(DB) 설계자"**
  * 방대한 메뉴와 제약 조건(야간 주문 불가 등) 데이터를 꼼꼼히 벡터화하고, 쿼리에 맞게 정확히 매핑하는 논리성이 필요합니다.

### 4. 🛎️ 컨시어지 (Concierge) + 🧠 NLU 자가 튜닝 (Self-Tuning)
* **담당 도메인:** 맛집 추천, 관광 안내, 짐 보관, 택시 호출 등 자유로운 질문 응대
* **담당 핵심 기술 스펙:** 
  * 관리자의 '수동 정정 데이터(오답 노트)' DB 적재
  * API 호출 시 이 정답 데이터를 **퓨샷(Few-shot)** 예시로 프롬프트에 동적 주입하는 로직 
* **적합한 성향 (Persona): "유연한 사고를 가진 문과 감성 크리에이터"**
  * 고객의 분방한 말투에 당황하지 않고, 인간적이고 친절한 톤앤매너로 퓨샷 예시를 구성하는 유연함이 어울립니다.

### 5. 🛠️ 시설 관리 (Facility) + 📚 RAG Ingestion (데이터 플라이휠)
* **담당 도메인:** 에어컨 누수, 변기 막힘, 조명 고장 등 시설 클레임 처리
* **담당 핵심 기술 스펙:** 
  * 심야 스케줄러(Cron)를 통한 미답변/에러 로그 질문 군집화(Clustering) 로직
  * 관리자 승인 시 텍스트를 즉시 임베딩하여 벡터 DB에 Insert 하는 지식 확장 파이프라인
* **적합한 성향 (Persona): "논리적인 트러블슈터 (문제해결사)"**
  * 수백 가지의 복잡한 증상 로그를 분석하고 원인별로 범주화(Clustering)하는, 날카롭고 분석적인 에러 추적 능력이 뛰어나야 합니다.

### 6. 🚑 긴급 대응 (Emergency) + 📝 NLG 데이터 요약 (인간 관리자 연계 배포)
* **담당 도메인:** 화재, 구급, 위협 등 **AI가 자체 판단해서는 안 되며 즉각적인 프론트 데스크(인간) 개입이 필수적인 상황의 전파**
* **담당 핵심 기술 스펙:** 
  * 패닉 상태인 고객의 횡설수설한 발화를 받아, 프론트 직원이 즉각 대처할 수 있게 **육하원칙 기반 핵심 팩트로 초고속 요약(NLG) 생성**
  * 요약된 데이터를 프론트 데스크 관리자 대시보드로 즉시 에스컬레이션(브리핑) 하여 쏘아주는 로직 프롬프트 튜닝
* **적합한 성향 (Persona): "위기 상황을 냉정하게 브리핑하는 특수 요원"**
  * 당황한 고객의 호출 속에서 "누가, 어디서, 통제가 필요한 문제는 무엇인지" 팩트만 명확하고 직관적으로 요약/전파하는 능력이 중요합니다.
