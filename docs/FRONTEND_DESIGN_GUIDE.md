# 🚀 Frontend AI Development & Component Assembly Guide

이 문서는 Anook 프론트엔드 팀이 **생성형 AI(Cursor, Claude 등)를 활용하여 페이지를 개발할 때 반드시 지켜야 하는 표준 가이드라인**입니다. 

우리의 프론트엔드 아키텍처는 **"레고 조립식(Component-Driven)"**으로 설계되었습니다. 개별 UI 블록(버튼, 카드, 모달 등)은 이미 완벽하게 구축되어 있으므로, 개발자와 AI의 역할은 **"기존 블록들을 알맞은 위치에 배치하고 연결하는 것"**에 집중되어야 합니다.

---

## 🏗 1. 핵심 철학: "절대 새로 만들지 말고, 조립만 하라"

새로운 페이지나 기능 레이아웃을 짤 때, AI에게 "게시판 테이블 만들어줘", "입력 폼 만들어줘"라고 지시하는 것은 **절대 금지**입니다. 중복된 CSS와 파편화된 컴포넌트가 양산되기 때문입니다.

### ✅ 컴포넌트 쇼케이스 (`/test`) 적극 활용
로컬 서버를 띄운 후 `http://localhost:3000/test` 에 접속하면 프로젝트에 존재하는 모든 UI 컴포넌트의 목록, 디자인, 그리고 **정확한 폴더 경로**를 확인할 수 있습니다.

**[올바른 작업 순서]**
1. `/test` 페이지에서 필요한 컴포넌트를 찾는다.
2. 컴포넌트 위에 적힌 파일 경로(예: `components/ui/Table/Table.tsx`)를 복사한다.
3. AI에게 **해당 경로의 컴포넌트들을 import 해서 배치해 달라고 구체적으로 명령**한다.

## 📏 2. 간격 및 여백 (Spacing) 규칙: 하드코딩 금지

디자인 시스템의 통일성을 위해 `margin`, `padding`, `gap` 등에 픽셀(px) 단위의 하드코딩을 절대 금지합니다. 모든 간격은 `src/styles/variables.css`에 정의된 **CSS 변수(Design Tokens)**를 사용해야 합니다.

### 📌 표준 간격 매핑 가이드
AI에게 프롬프트를 작성할 때, 아래의 규칙을 그대로 복사하여 지시하세요.

#### 1. 주요 레이아웃 간격 (Layout Spacing)
* **사이드바와 메인 화면 사이**: `48px` ➡️ **`var(--space-48)`** (App 최상위 컨테이너에서 사이드바와 콘텐츠 영역 구분)
* **좌우 전체 여백**: `32px` ➡️ **`var(--space-32)`** (화면 양쪽 끝에서 콘텐츠가 시작되는 지점)
* **상하 전체 여백**: `48px` ➡️ **`var(--space-48)`** (헤더 위쪽과 푸터 아래쪽의 여백)

#### 2. 컴포넌트 간 간격 (Component Gaps)
* **카드형 그리드 레이아웃**: `32px` ➡️ **`var(--space-32)`** (대시보드 내의 큰 섹션들이나 카드들 사이의 표준 간격)
* **섹션 내 요소 그룹**: `24px` ➡️ **`var(--space-24)`** (하나의 카드나 섹션 안에서 제목, 리스트, 푸터 등을 세로로 배치할 때)
* **아이콘과 텍스트 사이**: `12px` 또는 `6px` ➡️ **`var(--space-12)`** (사용자 프로필이나 메뉴 아이템 등)

#### 3. 내부 여백 (Padding)
* **큰 카드 컨테이너**: `32px` 또는 `48px` ➡️ **`var(--space-32)`** 또는 **`var(--space-48)`** (주요 정보가 담긴 카드의 내부 여백)
* **중간 크기 카드/리스트 아이템**: `24px` ➡️ **`var(--space-24)`** (채팅 목록, 지식 라이브러리 리스트 아이템 등)
* **버튼 및 입력창**: `24px`, `16px`, `12px` ➡️ **`var(--space-24)`, `var(--space-16)`, `var(--space-12)`** (터치 영역과 내부 여백)

#### 4. 세부 요소 간격 (Micro Spacing)
* **제목과 설명 사이**: `4px` 또는 `8px` ➡️ **`var(--space-4)`** 또는 **`var(--space-8)`** (강한 연관성을 가진 텍스트 간 아주 좁은 간격)
* **버튼 그룹 내 버튼 사이**: `12px` ➡️ **`var(--space-12)`** ('저장'과 '취소' 등 나란히 배치된 단추 간격)

## 🧩 3. 복잡한 도메인 컴포넌트 (RAG / AI)

가장 구현하기 까다로운 RAG(검색 증강 생성) 및 지식 관리(Knowledge Base) 컴포넌트들도 이미 내부 모달과 상태 관리 로직이 결합된 형태로 완성되어 있습니다.

* **`components/ui/Card/RagCandidateCard.tsx`**: AI가 지식을 찾지 못했거나 의도가 불분명할 때 띄우는 상담 후보 카드입니다. 상태(지식 없음/의도 불명)에 따른 뱃지 분기 처리가 포함되어 있습니다.
* **`components/ui/Knowledge/KnowledgeItem.tsx`**: 지식 베이스 리스트의 개별 아이템입니다. 아이템을 클릭했을 때 열리는 **상세 보기 모달과 수정 모달 로직이 이미 연결**되어 있습니다. 
* **`components/ui/Modal/ChatModal.tsx`**: 직원용 채팅 상담 모달입니다. `RequestCard`의 '상담 시작' 버튼과 이미 연동되어 있으므로 별도의 채팅창 구현이 필요 없습니다.

이 컴포넌트들은 내부 로직을 수정할 필요 없이, 단순히 화면에 배치(`render`)하기만 하면 됩니다.

## 🗣 4. AI 소통용 표준 프롬프트 템플릿 (Copy & Paste)

프론트엔드 개발 시, AI에게 레이아웃 조립을 맡길 때 아래 템플릿을 복사하여 사용하세요. 이렇게 지시해야 프로젝트 컨벤션에 맞는 코드를 생성해 줍니다.

```text
[역할]
너는 우리의 디자인 시스템과 컴포넌트 구조를 완벽하게 따르는 프론트엔드 개발자야.

[요청 사항]
새로운 [페이지 이름/용도] 페이지를 만들려고 해. 
절대 새로운 UI 요소를(div, span 떡칠 등) 새로 만들지 말고, 반드시 아래 명시된 기존 컴포넌트들을 import 해서 레고 조립하듯이 레이아웃만 구성해줘.

[사용할 컴포넌트 경로]
- components/layout/Sidebar.tsx
- components/ui/Table/Table.tsx
- components/ui/Card/SummaryCard.tsx
// (필요한 컴포넌트 경로를 /test 페이지에서 복사하여 추가)

[레이아웃 및 간격 규칙]
1. Sidebar는 왼쪽에 고정하고, 나머지 컨텐츠는 오른쪽에 가로 정렬해줘.
2. px 단위 하드코딩은 절대 금지! 모든 여백과 간격은 반드시 CSS 변수를 사용해줘.
   - Sidebar와 우측 컨텐츠 사이의 간격: var(--space-32)
   - SummaryCard 그룹과 Table 섹션 사이의 간격: var(--space-32)
   - 동일한 SummaryCard 끼리의 간격: var(--space-24)
3. 인라인 스타일보다 가급적 CSS Module(.module.css)을 사용해서 레이아웃 클래스를 만들어줘.
```

이 가이드를 통해 팀원 모두가 추후 원활한 디자인 수정을 위해 동일한 코드 품질과 일관된 UI/UX를 유지할 수 있도록 협조 부탁드립니다.
